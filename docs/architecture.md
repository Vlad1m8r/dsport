# Архитектура backend

## Общая схема
Backend — монолитное Spring Boot приложение, спроектированное
как доменно-ориентированное ядро, готовое к расширению.

Компоненты:
- REST API (Spring Web)
- JPA / Hibernate (PostgreSQL)
- Flyway (миграции)
- Telegram Mini App auth (initData)

## Архитектурный принцип
- Чёткое разделение:
    - domain (entities)
    - service (use-cases)
    - controller (API)
    - dto / mapper
- План (Template) и Факт (WorkoutSession) — разные модели.
- Минимум бизнес-логики в контроллерах.

## Почему не микросервисы
MVP ориентирован на скорость итераций и простоту:
- единая транзакционная модель,
- отсутствие распределённых транзакций,
- быстрые изменения схемы данных.

Миграция к микросервисам возможна позже (например, analytics / social).
